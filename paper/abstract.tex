\begin{abstract}
Existing geometry benchmarks measure whether language models can \emph{interpret} a given diagram and answer a question about it.
None measures whether a model can \emph{produce} a correct diagram from a description --- yet diagram production is increasingly the deployed task, as LLMs are asked to author worksheets, slides, and technical illustrations.
We introduce \GeoGenBench, the first benchmark for generative geometric reasoning with \textbf{fully-automatic, geometry-aware verification}.
\GeoGenBench pairs \textbf{801} natural-language prompts (e.g., ``draw an acute triangle $ABC$ with the altitude from $A$ meeting $BC$ at $H$'') with machine-checkable geometric properties that any correct rendering must satisfy (``$\angle AHB$ is right; $H$ lies on segment $BC$'').
Verification compiles the model's TikZ output to symbolic geometry via SymPy and decides each property in milliseconds, requiring no human grading or vision-language judge.
Across \textbf{four frontier models} (\opus, \sonnet, \haiku, \gpt) and \textbf{three generation strategies} on a stratified pilot, we find that \emph{the cost--quality Pareto frontier is dominated by cheap-model configurations}: \haiku with direct TikZ generation reaches 83\% strict pass at \dollars{0.016}/scenario, beating \opus with the same strategy (80\%, \dollars{0.328}) on both axes --- 3\pp more correct diagrams at $21\times$ lower cost.
A generic intermediate-representation strategy adds a further 4--20\pp on top.
We also report a verification-reliability audit: \cref{sec:hardening} closes 13 of 315 (4.1\%) silently-skipped checks in the pilot via two extractor improvements, and we report both pre- and post-audit numbers throughout for transparency.
We release the dataset, the procedural template engine that generated it (allowing reviewers to roll new contamination-free splits), the verification toolkit, and a public leaderboard.
\end{abstract}
